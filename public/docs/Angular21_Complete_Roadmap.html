Chapter 01

The Zoneless Era: Change Detection Re-imagined

Discover how modern Angular achieves type-safe reactivity and change detection without loading zone.js in the browser.

1.1 Zoneless Change Detection Setup

Historically, Angular relied on **Zone.js** to intercept asynchronous browser events (like click event handlers, timer intervals, or HTTP responses) and trigger global application change detection checks. Angular 21 stabilizes **Zoneless change detection**, enabling component-level checks only when reactives update.

👨‍🏫 Professor's Lecture: The Death of Zone.js

Zone.js worked by monkey-patching browser APIs (like addEventListener and setTimeout). While convenient, it caused performance issues. If an event fired in one component, Zone.js would run a dirty check across the *entire* application tree. Zoneless change detection, when paired with Signals, allows Angular to pinpoint exactly which node in the component tree requires a redraw, leading to faster execution times and reduced memory footprints.

app.config.ts
import { ApplicationConfig, provideExperimentalZonelessChangeDetection } from '@angular/core';

export const appConfig: ApplicationConfig = {
  providers: [
    provideExperimentalZonelessChangeDetection()
  ]
};
Chapter 02

Signals-Native Component APIs

Learn about signal-based inputs, output emitters, and model bindings that replace decorator-based configurations.

2.1 Signal Inputs, Outputs, and Models

Angular 21 replaces decorator-based component properties with signal-based alternatives, which compile down to standard reactive primitives.

signal-native.component.ts
import { Component, input, output, model } from '@angular/core';

@Component({
  selector: 'app-user-profile',
  standalone: true,
  template: `
    <p>Active User: {{ username() }}</p>
    <button (click)="updateStatus()">Toggle Active</button>
  `
})
export class UserProfileComponent {
  username = input.required<string>();
  isActive = model<boolean>(false);
  statusChanged = output<boolean>();

  updateStatus() {
    this.isActive.update(current => !current);
    this.statusChanged.emit(this.isActive());
  }
}
Chapter 03

Local State Synchronization with linkedSignal()

Learn about linkedSignal(), a new utility for synchronizing transient local state with upstream reactive inputs.

3.1 Dynamic State Resetting

A common UI challenge is resetting local component state (like a dropdown selection) when a parent input changes. The linkedSignal() API solves this problem by automatically resetting its value whenever a source signal changes.

linked-signal.component.ts
import { Component, input, linkedSignal } from '@angular/core';

@Component({
  selector: 'app-item-selector',
  standalone: true,
  template: `
    <h4>Category: {{ category() }}</h4>
    <p>Selected Item: {{ selectedItem() }}</p>
  `
})
export class ItemSelectorComponent {
  category = input.required<string>();

  selectedItem = linkedSignal({
    source: this.category,
    computation: (cat) => "Default " + cat + " Item"
  });

  selectItem(newItem: string) {
    this.selectedItem.set(newItem);
  }
}
Chapter 04

The Async Resource API: resource() & rxResource()

Learn about the resource() and rxResource() APIs, which allow you to fetch asynchronous data directly using signals.

4.1 Declaring Data Resources

Angular 21 introduces resource() and rxResource(), allowing you to fetch asynchronous data and manage loading states directly within the Signal reactivity model.

resource-api.component.ts
import { Component, signal, resource } from '@angular/core';

@Component({
  selector: 'app-product-profile',
  standalone: true,
  template: `
    <div>
      @if (product.isLoading()) {
        <p>Loading product...</p>
      } @else if (product.value()) {
        <h4>{{ product.value()?.name }}</h4>
        <p>Price: ${{ product.value()?.price }}</p>
      }
    </div>
  `
})
export class ProductProfileComponent {
  productId = signal(101);

  product = resource({
    request: () => ({ id: this.productId() }),
    loader: async ({ request }) => {
      const res = await fetch("https://api.store.com/products/" + request.id);
      return res.json();
    }
  });
}

Modern Signal Queries (viewChild, contentChild)

Learn about query functions that look up child components and elements as reactive signals.

5.1 Query Signal Functions

Angular 21 updates template child queries to use signals, providing automatic type checking and reflow safety.

queries.component.ts
import { Component, viewChild, ElementRef } from '@angular/core';

@Component({
  selector: 'app-modal-parent',
  standalone: true,
  template: `
    <div #modalContainer></div>
  `
})
export class ModalParentComponent {
  containerElement = viewChild.required<ElementRef<HTMLDivElement>>('modalContainer');
}
Chapter 06

SSR, Partial Hydration & Event Replay

Learn about partial hydration, which lets the browser download and hydrate specific page blocks only when needed.

6.1 Event Replay during SSR

To prevent loss of user actions (like clicks) that occur before the client-side JavaScript finishes downloading and executing, Angular 21 records those actions and replays them once the application is fully hydrated.

👨‍🏫 Professor's Summary: Web Performance Gains

By using **Zoneless change detection** and **Event Replay**, Angular 21 improves your site's core web vitals. It decreases the Time to Interactive (TTI) and reduces the First Input Delay (FID), creating a much faster and more responsive experience for your users.
